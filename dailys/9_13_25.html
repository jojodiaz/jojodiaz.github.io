Exploring std::expected

So let’s talk about std::expected. I learned about this C++23 change a couple days ago.

It takes the form of either:

template<class T, class E> class expected;

template<class T, class E> requires std::is_void_v<T> class expected;

Now this was my first time seeing the requires keyword (C++20), which probably means I haven’t spent enough time really learning C++ as I should. But here is a quick synopsis:

The requires keyword enforces certain characteristics upon a template class instantiation or a function.

To really understand this, we have to understand constraints and concepts (C++20).

  • Constraints are the specification of valid template arguments, aka what follows the requires keyword.
  • Concepts are named sets of these requirements.

This is a high-level explanation, and I should really do a deeper dive into the metaprogramming paradigm of C++ in general. But this is enough to excuse myself and further explain std::expected.

std::expected is a way to handle exceptions in a more type-safe and explicit manner.

For example, when we construct an std::expected value, we are allowed to either pass the expected type, or an std::unexpected instance containing the unexpected type. It's weird. But for example 

        
          std::expected<int, std::string> someParsingAction() {
              if(success) {
                return 0;
              }
              else
                return std::unexpected("[someObject::someParsingAction] Failed due to ...");
            }
